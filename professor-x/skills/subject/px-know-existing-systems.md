# px-know-existing-systems

## Purpose
Compare Professor X against nearby systems without overstating novelty.

## Knowledge
- AHE measures component-level harness ablations with a frozen model but does not provide DHE-style failure traces.
- MOSS validates source-level harness rewriting and motivates verify-then-commit.
- Darwin Godel Machine validates self-modifying agents but does not isolate harness contribution or consumer hardware constraints.
- Meta-Harness is a Lever 3 competitor using frontier APIs rather than local daily operation.
- Ratchet motivates outcome-driven skill retirement because unbounded skill growth can degrade retrieval.

## Use When
Use this skill for related-work claims, competitor analysis, and identifying what Professor X must measure to be defensible.
