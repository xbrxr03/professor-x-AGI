# px-know-self-evolving

## Purpose
Guide autonomous harness evolution through safe, measurable, reversible changes.

## Knowledge
- Evolution must use verify-then-commit: propose, apply in controlled workspace, check, accept or rollback.
- Reward-hacking includes weakening HIRO, bypassing policy, hardcoding task answers, faking success, or crediting no-op diffs.
- DHE should identify the failure layer before proposals are credited.
- BF tracks category-level behavior: tool-use, planning, and self-correction.
- LCAP should update from round-level evidence, not single-task noise.

## Use When
Use this skill for proposed harness changes, verification decisions, rollback analysis, or autonomous-run readiness.
