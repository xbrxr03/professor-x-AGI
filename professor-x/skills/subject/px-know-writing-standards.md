# px-know-writing-standards

## Purpose
Keep paper and project writing precise, cited, and useful for later publication.

## Knowledge
- Do not write result claims without experiment ids or citations.
- Mark missing data as `TODO(evidence)` instead of smoothing over it.
- Prefer compact mechanism diagrams, tables, and run summaries over broad narrative.
- Preserve limitations and failed runs because they make the benchmark credible.
- Use HIRO, DHE, BF, LCAP, MCA, ICS, and FED consistently with their definitions in `brain/inventions.md`.

## Use When
Use this skill for paper sections, daily updates, teaching notes, and synthesis.
