# Invention Research Workflow

## Purpose

This reference expands the core skill workflow for high-stakes invention work.

## Stage 1: Question framing

Prefer:

- one missing mechanism
- one target phenomenon
- one standard for novelty

Avoid:

- "find something revolutionary"
- "read everything and invent"
- broad trend summaries

## Stage 2: Local context inventory

Before web work, search the local project for:

- research notes
- benchmark docs
- hypothesis ledgers
- dead-end logs
- architecture constraints

The invention must fit the actual project substrate.

## Stage 3: Search strategy

Use at least four search lanes:

1. primary literature on the target phenomenon
2. adjacent literature that may already contain the mechanism
3. repositories or systems that operationalize related ideas
4. distant fields with structurally similar problems

## Stage 4: Candidate filtering

Promote only ideas that satisfy all of:

- addresses a real gap
- survives nearest-overlap comparison
- can be stated mechanistically
- implies at least one nontrivial prediction
- has a plausible path to measurement

## Stage 5: Output discipline

Always distinguish:

- literature fact
- inference from literature
- original synthesis
- speculative extension

This keeps the invention traceable.
