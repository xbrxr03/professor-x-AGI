# Phase 3 — Self-improvement loop (distillation flywheel) vs the wrong-edit ceiling

## Context (why now)
Phase 1+2 produced a sharp, evidence-backed conclusion: ALL harness mechanisms already exist
(edit-lever, anti-thrash, RLEF verifier-feedback retry), AND behavior-keyed retrieval showed that even
handing the model the correct fix barely helps (A5: +1 task, within noise; 9/14 fail WITH the answer).
Codex's taxonomy: **wrong-edit-verified-fail is 60-80% of failures.** So the binding constraint is
**model CAPABILITY — producing the correct edit** — not harness features or information. The only lever
left that attacks this is the **distillation flywheel** (Lever 1, un-paused after the 2026-06-22
corrupt-GGUF fix). Phase 3 drives it, with a TRUST GATE so improvement is verified and can't be gamed.

## Goal
A distilled qwen3:8b that produces correct edits MORE often than stock 8b on the wrong-edit failure
frontier — accepted ONLY if the gain GENERALIZES to held-out RENAMED anchors (the TGC trust gate), with
the train-vs-held-out Goodhart gap bounded.

## Two parallel, file-disjoint streams
Claude owns Rust (`src/`); Codex owns the Python training pipeline (`distill/`). No shared files.

### STREAM D — Claude (Rust: the TRUST GATE + collection quality)
The genuinely-new trust mechanism; buildable WITHOUT the GPU (so it proceeds while Codex trains).
- **TGC gate** in the repo-fix gate path (`src/main.rs` repo_fix_measure + the K-pass gate): accept a
  distilled model only if **held-out renamed-anchor pass@1 improves by ≥MDE (K-pass)**, not just the
  train set; log the **Goodhart gap** = train_pass@1 − anchor_pass@1.
- **NaN / stop-sanity guard** before any gate (from the corrupt-GGUF lesson): never gate a gguf that
  contains NaN or fails the stop-sanity one-shot.
- **Collection quality:** ensure `collect_trajectory` captures the TEACHER's verified-correct EDIT
  trajectories cleanly (native tool-call format) for the failure-frontier tasks.
- Milestones: D1 gate measures on the held-out anchor split + logs Goodhart gap; D2 NaN/stop-sanity
  guard rejects a bad gguf (test); D3 clean teacher-correct-edit corpus produced.

### STREAM E — Codex (Python: RUN the training pipeline on the failure frontier)
The pipeline already exists (`distill/run_after_reboot.sh`, `train_qlora.py`, llama.cpp convert+quant);
recipe fixes (assistant-only masking, alpha=2r) are in. Codex RUNS + targets the frontier — does NOT
rebuild the harness.
- Collect **teacher qwen3:14b** verified-correct trajectories on the corpus, weighted to the tasks the
  STUDENT fails (the wrong-edit frontier: hard set + families where 8b loses).
- QLoRA train student=8b (assistant-only loss masking + EPOCHS=2), merge, **quantize from f16 with a
  NaN check** (never ship a NaN gguf), serve as `profx-distilled-p3`.
- Milestones: E1 teacher frontier-corpus collected (N verified-correct); E2 QLoRA trained (loss curve) +
  clean GGUF (NaN-free) served; E3 hand `profx-distilled-p3` to Claude's gate.

## Sequencing / parallelism / GPU
- PARALLEL BUILD: Claude builds Stream D (Rust, no GPU) WHILE Codex runs Stream E (GPU: collect+train).
- INTEGRATION (sequential): Codex's `profx-distilled-p3` → Claude's TGC gate measures it on held-out
  renamed anchors → accept/reject, reported honestly.
- GPU rule (AGENTS.md): Codex owns the GPU during collection+training; Claude's gate MEASUREMENT runs
  AFTER, never concurrently (no two benches at once).

## Phase 3 success gate (falsifiable, pre-registered)
- **M3.1:** distilled held-out (renamed-anchor) pass@1 > stock 8b by ≥MDE (K-pass mean). 
- **M3.2:** Goodhart gap bounded (the train gain GENERALIZES) vs a train-only-gated baseline whose gap
  grows → demonstrates the TGC trust property.
- **KILL:** distilled doesn't beat stock on held-out → teacher/recipe insufficient at this scale →
  report honestly, fall back (don't ship a non-generalizing model). No fabricated wins.

## Fit
This is the moat (Phase 3 of PLAN_2026-06-21): trustable, local, self-improvement on a $400 GPU, where
"trust" = a Goodhart-proof held-out-renamed gate. It attacks the one thing Phase 1 proved is the
bottleneck (edit-production capability), and it's the first real test of TGC as a trust mechanism.
